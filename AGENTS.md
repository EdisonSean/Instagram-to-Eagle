# AGENTS.md

Guidance for coding agents working in this repository.

## Project Intent

`ins-eagle-sync` is a Windows-first Python CLI project. It downloads Instagram content through `gallery-dl`, stages files locally, reads metadata JSON, and imports files into Eagle through the Eagle Local API.

Do not build a GUI or Eagle plugin unless explicitly requested.

## Engineering Rules

- Keep the code modular and testable.
- Prefer small functions with clear inputs and outputs.
- Do not let downloader code, metadata parsing, Eagle API calls, and state management blur together.
- Do not write directly into an Eagle library. Downloads must go to the configured staging directory.
- Treat `imported.json` as the source of truth for duplicate prevention.
- Preserve Windows compatibility, especially paths and PowerShell scripts.

## Module Boundaries

- `cli.py`: command-line parsing and user-facing command orchestration.
- `config.py`: load and validate JSON configuration.
- `gallerydl_runner.py`: build and run `gallery-dl` commands.
- `metadata_parser.py`: normalize metadata generated by `gallery-dl`.
- `eagle_client.py`: Eagle Local API wrapper.
- `importer.py`: import workflow coordination.
- `state_store.py`: read/write imported state.
- `utils.py`: small pure helpers such as URL detection, title generation, and hashtag extraction.

## Testing

Add focused tests for pure logic first:

- URL mode detection
- Eagle title generation
- metadata normalization
- imported state behavior

Avoid tests that call Instagram, `gallery-dl`, or Eagle unless they are explicitly marked as integration tests.
